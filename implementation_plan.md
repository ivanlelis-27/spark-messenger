# Messenger PWA — Implementation Plan

A full-featured messaging app like Facebook Messenger, built as a Progressive Web App (PWA) using **Next.js 14 App Router**, **Supabase**, and **shadcn/ui**. Works natively on Android and iOS (via Add to Home Screen) without needing Xcode or the App Store.

### Phase 2 — Supabase Schema & Auth

#### [NEW] Database Schema (SQL migrations)

```sql
-- Profiles (extends auth.users)
create table public.profiles (
  id uuid references auth.users(id) primary key,
  username text unique not null,
  display_name text,
  avatar_url text,
  bio text,
  last_seen timestamptz default now(),
  created_at timestamptz default now()
);

-- Conversations (DM or group)
create table public.conversations (
  id uuid primary key default gen_random_uuid(),
  type text check (type in ('dm', 'group')) default 'dm',
  name text,           -- null for DMs, set for groups
  avatar_url text,
  created_by uuid references public.profiles(id),
  created_at timestamptz default now()
);

-- Participants (many-to-many)
create table public.participants (
  id uuid primary key default gen_random_uuid(),
  conversation_id uuid references public.conversations(id) on delete cascade,
  user_id uuid references public.profiles(id),
  joined_at timestamptz default now(),
  last_read_at timestamptz,
  unique(conversation_id, user_id)
);

-- Messages
create table public.messages (
  id uuid primary key default gen_random_uuid(),
  conversation_id uuid references public.conversations(id) on delete cascade,
  sender_id uuid references public.profiles(id),
  content text,
  type text check (type in ('text', 'image', 'audio', 'emoji')) default 'text',
  media_url text,
  reply_to uuid references public.messages(id),
  created_at timestamptz default now(),
  edited_at timestamptz
);

-- Reactions
create table public.reactions (
  id uuid primary key default gen_random_uuid(),
  message_id uuid references public.messages(id) on delete cascade,
  user_id uuid references public.profiles(id),
  emoji text not null,
  created_at timestamptz default now(),
  unique(message_id, user_id, emoji)
);

-- Push subscriptions (for Web Push)
create table public.push_subscriptions (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references public.profiles(id),
  subscription jsonb not null,
  created_at timestamptz default now()
);
```

#### RLS Policies (key ones)
- `profiles` — anyone authenticated can read; only owner can update
- `conversations` — only participants can read
- `participants` — only participants of the conversation can read
- `messages` — only participants of the conversation can read/insert
- `reactions` — only participants can read; owner can insert/delete

#### Auth Pages
- `app/(auth)/login/page.tsx` — Email + password form + Google OAuth button
- `app/(auth)/register/page.tsx` — Register form + auto-create profile
- `middleware.ts` — redirect unauthenticated users to `/login`

### Phase 6 — Push Notifications

#### User Review Required
> [!NOTE]
> Originally, the plan called for a Supabase Edge Function to trigger push notifications. To simplify local development and avoid requiring the Supabase CLI, we will instead trigger notifications internally using a Next.js API route (`/api/push`) immediately after a message is sent from the client. Let me know if you approve this simplified architecture.

#### [NEW] `public/sw.js`
- Manually written Service Worker to listen for `push` events and show notifications via `self.registration.showNotification`.
- Handles `notificationclick` events to focus the window or open the specific conversation.

#### [NEW] `components/pwa/PushManager.tsx`
- A client component added to the layout or settings page.
- Requests notification permission from the user.
- Registers `sw.js` and creates a PushSubscription.
- Saves the `PushSubscription` to the `push_subscriptions` table in Supabase.

#### [NEW] `app/api/push/route.ts`
- Next.js API route that receives a `conversation_id`, `message`, and `sender_name`.
- Fetches all `push_subscriptions` for participants in that conversation (excluding the sender).
- Uses `web-push.sendNotification()` to dispatch the push payload using the VAPID keys from `.env.local`.

#### [MODIFY] `components/chat/MessageInput.tsx`
- After successfully inserting a message to Supabase, calls `fetch('/api/push')` to trigger notifications for other participants.
